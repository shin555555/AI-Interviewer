# ロジック・計算式仕様

## 1. 属性スコア (Attribute Scores)
- 全10項目（P1〜P5, I1〜I5）をそれぞれ3問で判定。
- 各質問 3点(高) / 2点(中) / 1点(低) の合計（3〜9点）で計算。
- レーダーチャート表示用に正規化する。

## 2. メタデータ解析
- **回答時間 (Response Time):**
  - < 3s: 「直感的」（本心から回答しており迷いがない）
  - \> 15s: 「熟考」（迷いがある、または設問の理解に時間がかかっている）
- **修正回数 (Toggle Count):**
  - 選択肢を変更した回数を記録。自己認識の揺らぎを可視化。
- **矛盾判定 (Contradiction Check):**
  - 1属性内（3問）のスコアの分散（Standard Deviation）が大きい場合、「環境による変動あり」フラグを立てる。
- **自由記述テキスト (Qualitative Data):**
  - スコア計算（定量評価）には含めない。
  - 支援員が面談時に深掘りするための「定性データ」として保存。
  - AIの応答は受容のみ（「教えてくれてありがとうございます」「なるほど」等）。
- **再開キー (Session Key):**
  - 自動発行されたランダムな「4桁の数字」を使用。

## 3. 適職マッチング (MatchScore)
$$MatchScore = 100 - (\sum_{i=1}^{10} |U_i - R_i| \times \text{weight})$$

### 柔軟性 (I4) の反転ロジック
- ルーティンワークが重要な職種（データ入力、梱包等）では、**柔軟性スコアが低い = 適合度が高い** と判定する。
- 変化を嫌う特性を「ルーティンへの適性」としてプラス評価するため、計算時に $U_{I4}$ または要求値を反転させる。

## 4. 職種カテゴリーとマスターデータ
全40職種それぞれの10項目に対する「要求スコア（1〜5の数値）」は、最終的に **Googleスプレッドシート上の『職種マスターシート』** で管理し、システムはAPI経由で動的にこの基準値を読み込んで計算に使用する（これにより施設側での容易な微調整が可能）。

現在の初期マスターデータ（AI算出済みの40職種分の要求スコア表）については、`docs/specs/jobs_database.md` を参照すること。

1. **事務・IT** (10職種)
2. **物流・軽作業** (10職種)
3. **清掃・施設管理** (10職種)
4. **調理補助・サービス** (5職種)
5. **専門・その他** (5職種)

## 5. 出力テキスト要件 (やさしい日本語)
- 漢字の使用を抑え、中学生レベルの語彙にする。
- ネガティブな表現を避け、具体的な「できること」に変換する。
  - 例：「柔軟性が低い」→「決まった手順を正しく繰り返す力がある」
  - 例：「報連相が苦手」→「自分のペースで集中して作業に取り組める」
