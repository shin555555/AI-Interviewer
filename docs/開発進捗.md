## フェーズ13: 経験スコア加点マッチングロジックの実装 (2026年3月10日 完了)
*   **実装内容**: `jobMatcher.js` に `calculateExperienceScores(answers)` 関数を追加し、第2部（経験テスト）の回答データをマッチング計算に統合しました。
*   **新しい計算式**: `FinalScore = min(100, BaseScore + ExperienceBonus)` を採用。
    *   BaseScore: 従来通りの性格属性ベースのMatchScore（変更なし）
    *   ExperienceBonus: 職種のcategoryIdと一致するカテゴリーの経験スコア平均から算出（最大+15点）。スコア1（未経験・やりたくない）→ 0点、スコア5（プロ級）→ 15点の線形変換。
*   **後方互換性**: `calculateJobMatches()` の第2引数はデフォルト値 `{}` を持つため、管理者ダッシュボード等の既存の呼び出し元に変更不要。
*   **返却値の拡充**: 各職種の結果オブジェクトに `baseScore`・`experienceBonus` を追加し、スコア内訳の透明性を確保。

## フェーズ13: 第2部UI画面フローの実装 (2026年3月9日 完了)

*   **プログレスバー**: 「第1部 X/30」「第2部 X/10」と現在のパートが明示されるように更新しました。
*   **データ統合**: 第1部（30問）と第2部（10問）の回答データを統合した配列を `ResultScreen` に渡します。`calculateAttributeScores` は `attributeId` を持つ第1部の回答のみを使ってスコア計算し、第2部（`categoryId` のみ持つ）はスキップされます。GASへの保存時は全40問分のデータが送信されます。

## フェーズ14: 深掘り対話モードの実装 (2026年3月16日 UIコンポーネント実装完了)
本来の目的である「自己内省・心理的安全性」に回帰するため、適職診断とは別軸の「深掘り対話モード」を実装しました。LLM APIに依存せず、**レゴブロック式の動的合成**で毎回異なるパーソナライズされた対話を実現しています。

### 実装済みの内容
*   **`deepDiveData.js`（レゴブロック辞書）**: 24種タグ定義、3ルート×5フェーズの質問プール（計39問）、テンプレートブロック25件（共感・課題・リフレーミング）、リフレーミング辞書18パターン、トリセツ生成テンプレート20件、支援員代弁カード8パターン、ユーティリティ関数7本。
*   **`DeepDiveScreen.jsx`（チャットUI）**: 4つの入り口ルート（モヤモヤ / できたこと / トリセツ / お任せ）からの10ターン対話。タイピングインジケーター、フェーズ遷移時のテンプレートブロック自動挿入、ターン6-7でのフィーチャーUI（自由記述ハイライト）を実装。
*   **`DeepDiveResultScreen.jsx`（結果画面）**: 蓄積タグに基づく「私のトリセツ」（強み・環境・課題の3セクション）、リフレーミングカード、支援員への代弁シートを自動生成・表示。
*   **`App.jsx`統合**: ウェルカム画面に「深掘り対話をはじめる」ボタンを追加し、`deepdive` → `deepdive_result` の画面遷移ルーティングを実装。
