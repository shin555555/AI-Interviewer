# 開発進捗・ロードマップ (開発ストーリー)

このファイルは、単なるTODOリスト（`status.md`）の枠を超え、プロジェクトの過去の経緯、現在の立ち位置、そして未来のロードマップ（次に何をすべきか、なぜそれが必要か）を記述するためのものです。
AIエージェントはこのファイルを読み込むことで、「今どのフェーズにいて、今後どういった拡張やテストが控えているのか」という大局的なコンテキストを理解し、「すべてのタスクが終わったからやることがない」と誤認することを防ぎます。

- **フェーズ1: フロントエンド基盤構築**: React/Viteによるプロジェクト生成、CSS変数の定義、ルーティング設計。
- **フェーズ2: インタビューエンジン**: 30問の質問UI、ステップ形式、文字サイズ変更、セッション保持機能の実装。
- **フェーズ3: 解析・アルゴリズム**: スコア計算、MatchScoreロジック（柔軟性の反転ロジック含む）の実装。
- **フェーズ4: 職種データベース**: 40職種の静的マスターデータ作成。
- **フェーズ5: 結果表示・PDF出力**: Chart.jsによるレーダーチャート、やさしい日本語による解説出力、PDF化機能。
- **フェーズ6: 管理者ダッシュボードUI**: Chart.jsによる全体統計表示（ドーナツ、棒グラフ等）、個別ユーザー時系列データのプロットUI。
- **フェーズ7: GASバックエンド連携**: Google Apps Scriptを利用し、フロントエンドとスプレッドシート間でAPI通信する基盤。`Code.gs`の作成、フロントエンド(`spreadsheetAPI.js`, `App.jsx`, `ResultScreen.jsx`)でのAPI連携・POST送信の実装が完了した。
- **フェーズ8: UX/UIのブラッシュアップと微調整 (現在進行中)**: 例外エラー処理、スマホ実機でのUI調整、スプレッドシートマスター側の動作確認を行う。

GAS連携によるE2Eテストが無事に完了し、フロントからスプレッドシートへのデータ疎通が確認できました。
現在、実運用を見据えたUI/UXの最終的なブラッシュアップ（エラー時のリカバリー表示や、モバイルでのレスポンシブな操作感の微調整）を行うフェーズに入っています。

## フェーズ13: 第2部UI画面フローの実装 (2026年3月9日 完了)
*   **実装内容**: `experiencePool.js` に `selectRandomExperienceQuestions()` 関数を追加し、`InterviewScreen.jsx` を2部構成フローに対応する形に拡張しました。
*   **画面フロー**: 第1部（性格テスト30問）完了後、チャットUI内でシームレスにトランジションメッセージを表示し、「続ける」ボタンで第2部（経験テスト10問）へ移行します。専用の中間画面を挟まず、同一のチャットUIの流れの中で自然に切り替わるUX設計となっています。
*   **トランジションメッセージ**: 第1部完了時に「前半のテストが終了しました。ここまでお疲れさまでした。」「ここからは、これまでの経験や趣味について10問だけ質問させてください。気軽に答えていただければ大丈夫です。」をAIメッセージとして表示します。
*   **プログレスバー**: 「第1部 X/30」「第2部 X/10」と現在のパートが明示されるように更新しました。
*   **データ統合**: 第1部（30問）と第2部（10問）の回答データを統合した配列を `ResultScreen` に渡します。`calculateAttributeScores` は `attributeId` を持つ第1部の回答のみを使ってスコア計算し、第2部（`categoryId` のみ持つ）はスキップされます。GASへの保存時は全40問分のデータが送信されます。
*   **中断・再開対応**: `waitingForPart2Start` 状態もLocalStorageに保存されるため、第1部と第2部の間で中断してもセッション再開が可能です。
*   **GitHub Pagesへのデプロイ**: 実装完了後、`npm run deploy` によりGitHub Pages (`https://shin555555.github.io/AI-Interviewer/`) へ反映済みです。
