# UI ワイヤーフレーム仕様書

本ドキュメントは「AI自己理解サポーター『ワーク・プロファイル』」の主要な画面構成（ワイヤーフレーム）およびUI/UXの設計方針を定義します。

---

## 1. 利用者向け：インタビュー画面（PC基本・レスポンシブ想定）

### 概要・コンセプト
*   **心理的安全性**: 評価されているプレッシャーを減らす「対話型（チャット）UI」。
*   **プロフェッショナルなトーン**: 「やさしい日本語」を用いつつ、一人の職業人として扱う。子供扱いや「〜だね」「押してね」などのカジュアルな語尾は一切禁止し、「〜です・ます」「〜してください」を徹底する。
*   **PCファースト設計**: 基本の入力端末はPC（マウス操作）を前提とするが、タブレットやスマホでも崩れないレスポンシブ設計とする。

### UIアクション定義
*   **チャットエリアの幅制限**: PCモニター上でもテキストが横に長くなりすぎないよう、中央のチャットエリアの最大幅（`max-width: 800px;`程度）を固定し、読みやすい視線移動を保つ。
*   **選択肢ボタン**: マウスでクリックしやすい大きなボタン（PC表示時は横並び、スマホ等では縦積み）。
*   **自動スクロール**: 利用者がアクション（ボタン選択やテキスト送信）をするたびに、画面下部（最新のやり取り）へスムーズにスクロールする。

## 2. 管理者向け：ダッシュボード画面（PCブラウザ想定）

### 概要・コンセプト
*   **経営・運営の意思決定支援**: データを「表」から手動でグラフ化する手間を完全に排除する。
*   **ファーストビューで結論提示**: 開いた瞬間に事業所の全体傾向（強み、課題、職種トレンド）を自動グラフで把握できるようにする。

### UI機能定義
*   **BIダッシュボード化**: スプレッドシート側のデータ（APIレスポンス等）を受け取り、フロントエンド側のグラフ描画ライブラリ（Chart.js等）によって各種グラフを自動描画する。
*   **定性データの参照性**: 下部の利用者一覧テーブルにおいて、「自由記述」欄を一覧で確認可能とする。面談や支援計画作成時の定性フィードバック材料として即座に活用できる。
