# AI-Interviewer フェーズ13: 経験テスト（第2部構成）UI実装ガイド

## 1. タスクの背景と目的
これまでの開発で、AIワープロファイルは「性格テスト（全30問）」を行うアプリケーションとして完成しました。
しかし、マッチング精度をさらに向上させるため、**フェーズ13として「経験・スキルチェック（全10問）」の第2部テストを追加**することになりました。

すでに裏側となる質問のデータベース (`app/src/data/experiencePool.js`) は作成されています。
あなたのタスクは、**「性格テスト（第1部）」完了後、シームレスに「経験テスト（第2部）」へ移行するUI・画面フローを実装する**ことです。

## 2. ゴールとなる画面遷移フロー
現在のフロー：
`WelcomeScreen` → `名前入力` → `InterviewScreen (性格テスト30問)` → `ResultScreen (結果)`

**変更後のフロー：**
1. `WelcomeScreen` → `名前入力`
2. **`InterviewScreen` (第1部: 性格テスト30問)**
3. **トランジション (画面上での案内)**:
   - 30問完了時に「前半のテストが終了しました。」「ここからは、これまでの経験や趣味について10問だけ質問させてください。」というようなAIからのシステムチャット・案内を表示させる。（専用のクッション画面を挟むか、チャット内でシームレスに続けるかは実装方針にお任せします。チャットUIを活かすとUXが良いです）。
4. **`ExperienceScreen` または `InterviewScreenの再利用` (第2部: 経験テスト10問)**
   - `experiencePool.js` からランダムに10問を抽出して出題。
   - UIのトーン（やさしい日本語、チャット形式、文字サイズ変更など）は第1部と完全に共通させる。
5. **集計処理・データ統合**
   - 第1部（性格30問）の回答データと、第2部（経験10問）の回答データを一つにまとめる。
6. `ResultScreen (結果)`
   - まとめたデータをResultScreenへ渡して処理する。※現在の `ResultScreen` はとりあえず第1部のデータが正しく入っていれば動作する構造ですが、第2部のデータもバックエンド(GAS)には保存させたいため、統合した配列データを渡す構想です。

## 3. 関連ファイル
以下のファイルを `view_file` で読み込んで現状の実装を把握してください。

*   **`app/src/App.jsx`**: 画面遷移のメインコントローラー（`screen` stateで管理）。現在は `welcome` | `interview` | `complete` | `admin` の状態を持っています。
*   **`app/src/components/Interview/InterviewScreen.jsx`**: 現在のインタビュー画面。
    *   内部で `const QUESTIONS = useMemo(() => selectRandomQuestions(3), [])` として、質問データを直接インポート・生成しています。
*   **`app/src/data/experiencePool.js`**: 経験テスト用の質問データ。ここにはすでに5カテゴリー×複数問のデータがあり、これを使ってランダム10問を抽出するロジック（または関数）を使用/実装してください。
*   **`app/src/components/Result/ResultScreen.jsx`**: （参考程度。最終的な `answers` データの受け皿）

## 4. 推奨される実装アプローチ（React設計）

現在の `InterviewScreen.jsx` は性格テスト（`questionPool.js`）に密結合しています。これに対応するためのアプローチは大きく2つあります。やりやすい方を選択してください。

**アプローチA: `InterviewScreen` のコンポーネント汎用化（おすすめ）**
`InterviewScreen.jsx` をリファクタリングし、`questions` (出題する配列) や `onComplete` などをPropsとして受け取る汎用コンポーネントにする。
`App.jsx` 側で「第1部用」と「第2部用」でPropsを切り替えたり、中間の画面遷移を管理する。

**アプローチB: `ExperienceScreen.jsx` の新規作成**
`InterviewScreen.jsx` をコピーして `ExperienceScreen.jsx` を作成し、内部の読み込み先を `experiencePool.js` に変更する。
`App.jsx` では `interview-part1` → `interview-part2` という画面遷移Stateを追加する。

## 5. UI/UXの制約事項 (絶対に守ること)
- デザインのトーン、やさしい日本語のルール、Professionalな扱いは維持すること。
- 「〜だね」などの子供扱いの表現はNG。
- スマホ対応（レスポンシブ）が崩れないようにすること。
- 現在のGAS保存機能（回答データの通信）を壊さないこと。第1部・第2部の回答データを一緒の配列（または構造体）にして `ResultScreen` の `answers` に流せるようにしてください。
