# Google スプレッドシート & GAS セットアップガイド

このガイドでは、「ワーク・プロファイル」のバックエンドとなる **Googleスプレッドシート** と **Google Apps Script (GAS)** の設定方法を説明します。

---

## 🔧 ステップ1: Googleスプレッドシートの新規作成

1. [Google スプレッドシート](https://sheets.google.com) にアクセスします。
2. 「空白のスプレッドシート」を新規作成します。
3. スプレッドシート名を **「ワーク・プロファイル データ管理」** などに変更します。

---

## 📝 ステップ2: GASスクリプトの貼り付け

1. スプレッドシートのメニューから **「拡張機能 → Apps Script」** をクリックします。
2. Apps Script エディタが開きます。
3. デフォルトの `Code.gs` の中身を **すべて削除** します。
4. 本プロジェクトの `gas/Code.gs` ファイルの中身を **すべてコピー&ペースト** します。
5. 保存します（`Ctrl + S`）。

---

## 🚀 ステップ3: 初期セットアップ（シートの自動生成）

1. Apps Script エディタ上部の **関数セレクトボックス** で `setupAll` を選択します。
2. **「▶ 実行」** ボタンをクリックします。
3. 初回実行時、**Googleアカウントの権限承認** ダイアログが表示されます：
   - 「権限を確認」をクリック
   - 自分のGoogleアカウントを選択
   - 「詳細」→「安全ではないページに移動」をクリック（開発段階のため）
   - 「許可」をクリック
4. 実行が成功すると、スプレッドシートに以下の **3つのシート** が自動作成されます：

| シート名 | 内容 |
|---|---|
| **回答データ** | 利用者の診断結果を1行1件で蓄積する（ヘッダー自動設定済み） |
| **職種マスター** | 40職種の要求スコア初期データ投入済み（施設側で自由に編集可能） |
| **設定** | 管理者パスワード、事業所名などの設定値 |

---

## 🌐 ステップ4: WebアプリとしてGASをデプロイ

1. Apps Script エディタから **「デプロイ → 新しいデプロイ」** を選択します。
2. デプロイタイプで **「⚙ ウェブアプリ」** を選択します。
3. 以下の設定を行います：
   - **説明:** `ワーク・プロファイル API v1`
   - **次のユーザーとして実行:** `自分`
   - **アクセスできるユーザー:** `全員`  
     ※ フロントエンドからのアクセスを可能にするため
4. **「デプロイ」** をクリックします。
5. 表示された **ウェブアプリURL** をコピーします。

   ```
   例: https://script.google.com/macros/s/AKfycb.../exec
   ```

---

## 🔗 ステップ5: フロントエンドとの接続

1. プロジェクトの `app/` ディレクトリにある `.env.example` ファイルをコピーして `.env` ファイルを作成します（既に存在する場合は編集）。

   ```bash
   cp app/.env.example app/.env
   ```

2. `.env` ファイルを編集し、ステップ4でコピーしたURLを設定します：

   ```env
   VITE_GAS_API_URL="https://script.google.com/macros/s/あなたのデプロイID/exec"
   VITE_ADMIN_PASSWORD="admin1234"
   ```

3. フロントエンドの開発サーバーを再起動します。

   ```bash
   cd app
   npm run dev
   ```

## ✅ ステップ6: 動作確認

### API テスト（ブラウザで直接確認）

以下のURLをブラウザに貼り付けて、JSONレスポンスが返ることを確認します：

```
https://[あなたのデプロイURL]?action=getUsers
https://[あなたのデプロイURL]?action=getJobsMaster
```

### フロントエンドからの書き込みテスト

1. フロントエンドでインタビューを最後まで完了します。
2. スプレッドシートの「回答データ」シートに新しい行が追記されているか確認します。

### 管理ダッシュボードでの読み込みテスト

1. フロントエンドの管理者ダッシュボードにログインします。
2. スプレッドシートから取得されたデータがグラフ等に反映されるか確認します。

## ⚠️ 注意事項

### コードを更新した場合

GASのコードを更新した後は、**必ず新しいバージョンとして再デプロイ** してください。

1. 「デプロイ → デプロイを管理」
2. 鉛筆マーク（編集）をクリック
3. バージョンを **「新しいバージョン」** に変更
4. 「デプロイ」をクリック

> ⚠️ URLは変わりませんが、バージョンを更新しないとコード変更が反映されません。

### CORS（クロスオリジン）について

GASのウェブアプリは自動的にCORSヘッダーを付与するため、通常は問題ありませんが、`fetch` 時には以下の点に注意してください：

- `mode: 'no-cors'` は使用しない（レスポンスが読めなくなる）
- GASのPOSTでは `Content-Type: application/json` ではなく `text/plain` で送信する必要がある場合があります

### スプレッドシートの共有設定

事業所内の複数スタッフがスプレッドシートを閲覧できるよう、適切に共有設定を行ってください。

## 📂 シート構成図

```
「ワーク・プロファイル データ管理」スプレッドシート
├── 📊 回答データ
│     ├── A: タイムスタンプ
│     ├── B: 利用者ID
│     ├── C: 利用者名
│     ├── D: 再開キー
│     ├── E〜N: 10属性の生スコア (P1〜I5)
│     ├── O〜X: 10属性の正規化スコア (1〜5)
│     ├── Y: 合計回答時間(ms)
│     ├── Z: 確信度
│     ├── AA: トグル回数
│     ├── AB: 矛盾フラグ
│     ├── AC〜AG: 強み・課題・推奨職種・カテゴリー
│     ├── AH: 自由記述テキスト
│     └── AI〜: 30問×5項目 = 150列の個別回答データ
│
├── 🏢 職種マスター
│     ├── A: ID (job_01〜job_40)
│     ├── B: 職種名
│     ├── C: カテゴリーID
│     ├── D: カテゴリー名
│     └── E〜N: 10属性の要求スコア (1〜5)
│
└── ⚙ 設定
      ├── A: 設定項目
      ├── B: 値
      └── C: 説明
```
