# コマンド概要
`/sync` コマンドは、セッション終了時やキリの良いタイミングで、ドキュメント類の完了・未完了状態を「現在のソースコードの実際の状態」に一致させる保守用ワークフローです。
このコマンドにより、「コードは書いたが `status.md` にチェックを入れ忘れた」といった事態を防ぎ、次回の `/onboard` を正確なものにします。

# 実行手順

1. `status.md` (c:\Users\user\Desktop\AI-Interviewer\docs\status.md) および `開発進捗.md` (c:\Users\user\Desktop\AI-Interviewer\docs\開発進捗.md) 、ならびに `深掘り対話モード実装計画書.md` (c:\Users\user\Desktop\AI-Interviewer\深掘り対話モード実装計画書.md) を読み込み、現在の進捗状況、未チェック（`[ ]`）の項目、今後のロードマップを把握する。
2. これまでに作成・変更した実装ファイル（例：`app/src/`内のReactコンポーネント、または関連するCSS）を必要に応じて参照し、本当にその機能が存在するか確認する。
3. 機能が完成しているにも関わらず未チェックになっている場合、`replace_file_content` ツールを使用してチェックボックス（`[x]`）へ確実に入力（更新）する。
4. ドキュメント（要件定義、実装計画など）に追記漏れの決定事項があれば、各ファイルにも追記を行う。
5. ユーザーに対し、「ドキュメントとコードの同期が完了し、`status.md` が最新の状態にアップデートされました」と報告すること。
