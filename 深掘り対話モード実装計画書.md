# 深掘り対話モード 実装計画書 (Deep Dive Mode Implementation Plan)

## 1. 概要 (Goal Description)
「適職診断」とは異なるアプローチで、利用者が自身の強みや課題に対する自己理解を深める「深掘り対話（自己理解）モード」を新規に実装する。
生成AI（LLM）APIを使用せず（無料）、あらかじめ定義した対話シナリオのパーツ（レゴブロック）と、利用者の選択・自由記述による入力キーワードを動的に組み合わせることで、毎回異なる深くパーソナライズされた対話体験を提供する。

## 2. コア・コンセプト (心理学的アプローチ)
本モードは `インタビュー概念.md` に定義された以下の原則をシステム（レゴブロックの辞書および分岐設計）の根幹とする。
*   **ストレングス・ベースド & スモール・ウィン**: 利用者がネガティブな選択をした場合でも、最終的には「できていること（強み）」に変換（リフレーミング）して肯定する着地を必須化する。
*   **「なぜ」の回避と水平展開**: 理由を直接問う「なぜ？」を避け、「どんな時に？」「何がきっかけで？」というSTAR法（状況や行動）を引き出す柔らかい言い換えを辞書パーツの基本トーンとする。
*   **ナラティブ・セラピー（外在化）**: 課題やモヤモヤを「自分自身の欠点」としてではなく、「外部のキャラクターや状況」として客観視できるようなセリフ設計（例：「〇〇さんを疲れさせる原因は…」）を行う。

## 3. アーキテクチャ・動作ロジック
### 3.1 漏斗（じょうご）型シナリオ構造
入り口は複数用意するが、最終的には一つの「トリセツ（取扱説明書）」生成へ向けて収束する構造とする。
*   **入り口のランダム化**: 「モヤモヤする」「できたこと」「トリセツ作成」「お任せ」等の入り口（テーマボタン）を用意し、各テーマの中でも初回の質問文はランダムに抽出する。
*   **10ターン目安**: 約10往復の対話（ターニングポイントを持つ）で一回のセッションを終える構成とする。

### 3.2 レゴブロック式動的合成・変数代入
*   **タグ付けシステム**: 利用者が選択した選択肢それぞれに「環境過敏」「ルーティン重視」などの裏タグ（属性）を設定し、蓄積する。
*   **テンプレート合成**: タグの組み合わせに応じて、「共感ブロック」「課題ブロック」「リフレーミング（強み変換）ブロック」を連結し、一つの返答文にする。
*   **自由記述の代入**: 選択肢だけでなく、ユーザーが自由記述欄に入力した「生きた言葉」を変数として記憶し、その後のAIのテンプレートの `[ユーザーキーワード]` 部分に直接代入することで、没入感と共感性を高める。

### 3.3 【将来拡張】過去記録との連携（コンテキスト継承）
初期実装後（次期フェーズ）の拡張として、LocalStorageやスプレッドシートに保存された前回・過去の診断結果（タグや選ばれたキーワード）を読み込み、「先週は〇〇だと仰っていましたが…」というように、過去の文脈を引き継いで対話を再開・深化できるアーキテクチャを想定してシステムを構築する。

## 4. UI/UX設計
*   **チャット形式UI**: 「AIの吹き出し」と「自分の返信」が並ぶLINEのようなインターフェース。
*   **入力形式 (ハイブリッド)**:
    *   基本は選択肢ボタン（3〜4つ）による進行。
    *   各ターンで、選択肢の下に「薄めのトーンで控えめに『💬 自分の言葉で書き足す（任意）』」といった自由記述欄を配置（いつでも補足可能）。
    *   「ここぞ」という深掘り（ハシゴを登り切った価値観の引き出し）タイミング（例: 7ターン目など）では、**選択肢を非表示（または最小化）にし、自由記述欄を画面中央でピカッとハイライト（大きく目立たせる）するフィーチャーUI**を採用する。メリハリのあるコントラストによって、利用者の「自分の言葉」を引き出す。
*   **最終出力**: AIとの対話履歴データに基づき、以下4つの要素を統合した最終レポート（PDF出力対応）を自動生成し提示する。
    1. **私の取扱説明書（トリセツ）**: 自分が心地よく働ける環境や、力を発揮しやすい条件を言語化したもの。
    2. **ネガティブのポジティブ変換カード（リフレーミング）**: 「〜が苦手」から「〜という素晴らしい才能・強みがある」への変換など、自己肯定感を高めるメッセージ。
    3. **「できた！」の発見レポート（スモール・ウィン）**: 苦手な状況下の中でも通所できた等、日常の隠れた実績や軌跡に気づかせるまとめ。
    4. **支援員さんへの代弁シート**: 本人が直接言いにくい配慮事項を、「こんな風に伝えてみませんか？」とAIが代弁して提案するカード（支援員に見せる用）。

## 5. スプレッドシート（GAS）連携
対話パターンの辞書（レゴブロックデータ）、リフレーミング辞書は初期実装としてはフロントエンド内のJS/JSONに持つが、後続フェーズでGoogleスプレッドシートから取得・更新できる設計とし、支援員が後から質問やパターンの追加を行えるように保つ。
